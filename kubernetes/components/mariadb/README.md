# MariaDB Component

Provides a per-app MariaDB database, user, and grant on the shared MariaDB instance.

## Variants

- `app` — Creates Database/User/Grant CRs and a connection Secret
- `app-template` — Same as `app` plus patches HelmRelease to inject the password via `valuesFrom`

## Usage

In your app's `app.ks.yaml`:

```yaml
spec:
  components:
    - ../../../../components/mariadb/app
  dependsOn:
    - name: mariadb-operator
      namespace: database
  postBuild:
    substitute:
      APP: my-app
      # Optional overrides:
      # DATABASE_OWNER: custom-user    # defaults to ${APP}
      # MARIADB_INSTANCE: mariadb      # name of the MariaDB CR in database ns
```

## What gets created

In the **database** namespace:
- `Database` CR named `${APP}`
- `User` CR named `${APP}` (password auto-generated into `${APP}-mariadb-password` secret)
- `Grant` CR giving ALL PRIVILEGES on the database to the user

In the **app's** namespace:
- `Secret` named `${APP}-mariadb` with `host`, `port`, `database`, `username`, `url` keys

## Password

The password is auto-generated by the mariadb-operator (`generate: true` on the User CR).
The generated secret `${APP}-mariadb-password` lives in the database namespace.
The `app-template` variant injects it into the HelmRelease via `valuesFrom`.

For apps that need the password as an env var, reference the secret directly:
```yaml
env:
  DB_PASSWORD:
    valueFrom:
      secretKeyRef:
        name: ${APP}-mariadb-password
        key: password
```
